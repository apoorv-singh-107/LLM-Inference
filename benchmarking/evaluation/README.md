# Model Evaluation Framework

This directory contains the scripts responsible for the analytical evaluation of benchmark results produced by the scripts in the `../testing` directory. The framework is designed to provide a holistic assessment of a model's quality by combining deterministic metrics, LLM-based judgments, and procedural correctness scores.

## Evaluation Workflow

The evaluation process is orchestrated by `evaluate_benchmark.py` and leverages specialized modules to score different facets of a model's performance.

### `evaluate_benchmark.py`

This is the main driver of the evaluation pipeline. It takes the raw output from benchmark runs and synthesizes them into a final, ranked comparison of models.

**Key Responsibilities**:

1.  **Data Loading**: Loads benchmark results and ground truth answers from the Excel files generated by the testing scripts.
2.  **Deterministic Metrics**: Computes objective, "Layer 1" metrics such as:
    -   Response Speed (Total Time, Time to First Token).
    -   Reliability (`Stream Ended Cleanly` percentage).
    -   Token Efficiency (Tokens/Second).
3.  **LLM-as-a-Judge (Layer 2)**: For subjective quality assessment, this script calls an external, powerful LLM (like Gemini) to act as a "judge." The judge scores each model's response against the ground truth on several criteria:
    -   `Accuracy`
    -   `Completeness`
    -   `Relevance`
    -   `Hallucination-Free`
    It uses specialized prompts for regular and Out-of-Knowledge-Base (OOKB) questions and caches results in `judge_cache.json` to avoid redundant API calls.
4.  **Agentic Correctness Integration**: It imports and runs `agentic_eval.py` to score the correctness of the model's tool-usage patterns.
5.  **Final Ranking**: It calculates a final `composite_score` for each model by combining the scores from all evaluation layers using a weighted formula. This score is used to produce a definitive ranking.
6.  **Exporting**: The script generates a comprehensive, multi-sheet `evaluation_results.xlsx` file containing the final rankings, detailed scores, and breakdowns by complexity.

### `agentic_eval.py`

This module focuses on evaluating the **correctness of the model's decision-making process** (i.e., its agentic behavior), not just the final answer. It enforces a set of rules about which tools should be used for different types of questions.

**Key Features**:

-   **Rule-Based Scoring**: Defines a clear set of rules for tool routing. For example:
    -   Financial questions must use `querifai` first.
    -   Out-of-knowledge-base (OOKB) questions should escalate to `web_search` or be refused directly, not get stuck in a `querifai` loop.
    -   Multi-source questions must use both `querifai` and `docufai`.
-   **Violation Detection**: The script parses the sequence of tool calls from the benchmark logs and identifies specific violations of these rules (e.g., `WEB_SEARCH_BEFORE_RAG`, `MULTI_SOURCE_MISSING_QUERIFAI`).
-   **Scoring**: A penalty is applied for each violation, resulting in an `agentic_score` (from 0 to 5) that reflects how logically the model behaved.

## Files

-   `evaluate_benchmark.py`: The main evaluation script.
-   `agentic_eval.py`: The module for scoring tool-use correctness.
-   `judge_cache.json`: A cache file to store results from the LLM-as-a-Judge to reduce costs and speed up re-runs.
-   `evaluation_results.xlsx`: The final, detailed report generated by the evaluation process.
