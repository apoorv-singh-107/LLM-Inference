# Benchmarking and Evaluation Framework

This is a custom module, will publish more on what and how it needs to be used but till then you can take a read if you want.

This directory contains a comprehensive framework for benchmarking the performance of various Small Language Models (SLMs) and evaluating their quality across multiple dimensions. The framework is split into two core components: `testing` and `evaluation`.

## Overall Workflow

The end-to-end process for evaluating a set of models follows two main stages:

1.  **Testing**: First, use the scripts in the `testing/` directory to execute a standardized set of questions against one or more model endpoints. These scripts measure performance metrics and record the models' raw responses and agentic actions. This stage produces one or more `benchmark_results.xlsx` files.

2.  **Evaluation**: Next, use the scripts in the `evaluation/` directory to analyze the data collected during the testing phase. These scripts score the models on quality, correctness, and adherence to procedural rules, ultimately producing a final, ranked report.

## Subdirectories

### `testing/`

This subdirectory contains all the tools required to **run benchmarks and tests**. Its purpose is to gather the raw performance data and model outputs.

-   **Key Scripts**:
    -   `benchmark_runner.py`: A configurable script to run questions against the `/generative_response` endpoint and record metrics like latency, throughput, and tool usage.
    -   `test_gateway.py`: A robust suite for testing the infrastructure of the LLM gateway, including security, load, and concurrency tests.
    -   `querfai/benchmark_runner.py`: A specialized version of the benchmark runner for deep evaluation of the `querifai` sub-agent's SQL generation capabilities.

For more details, see the `README.md` inside the `testing/` directory.

### `evaluation/`

This subdirectory contains the tools to **analyze and score the results** generated by the `testing` scripts. Its purpose is to transform raw data into actionable insights and model rankings.

-   **Key Scripts**:
    -   `evaluate_benchmark.py`: The main orchestrator that computes deterministic metrics, runs an "LLM-as-a-Judge" quality assessment, and integrates scores to produce a final ranking.
    -   `agentic_eval.py`: A specialized module that scores how correctly a model uses tools, based on a predefined set of rules.

For more details, see the `README.md` inside the `evaluation/` directory.
