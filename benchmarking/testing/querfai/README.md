# Querifai Benchmark Runner

This directory contains a specialized benchmark runner focused on evaluating the `querifai` sub-agent. The script is tailored to capture and analyze the unique aspects of `querifai`'s behavior, particularly its SQL generation capabilities.

## Files

-   `benchmark_runner.py`: The main script for running the benchmark.
-   `slm_exploration.xlsx`: The input Excel file containing the questions to ask the model. This file also includes an "SQL Query" column with the expected ground-truth SQL for comparison.

## `benchmark_runner.py`

This script is an advanced version of a general benchmark runner, with special features for deep-diving into `querifai`'s performance.

### Key Features

-   **SSE Stream Parsing**: Connects to the `/generative_response` endpoint and parses the Server-Sent Events (SSE) stream in real-time.
-   **Detailed Querifai Tracing**: It specifically listens for `brain/querifai/text` and `brain/querifai/body` events to track every invocation of the `querifai` sub-agent.
-   **Step-by-Step Tool Call Capture**: Within each `querifai` invocation, it captures the precise sequence of internal tool calls (e.g., `get_tables`, `execute_mysql_query`) and their corresponding results. This is achieved by tracking unique `call_id`s throughout the stream.
-   **SQL Generation Analysis**: A primary feature is its ability to extract the SQL queries generated by the `execute_mysql_query` tool.
-   **Multi-Sheet Excel Reports**: It saves the results to `benchmark_results.xlsx` with highly detailed, formatted sheets:
    -   `Summary`: A high-level overview of the benchmark run.
    -   `Benchmark Results`: One row per question with overall performance metrics.
    -   `Querifai Details`: One row per `querifai` invocation, detailing its duration, token usage, and final output.
    -   `Querifai Steps`: The most granular view, with one row for every single tool call within every invocation, including arguments and results.
    -   `SQL Comparison`: A dedicated sheet that compares the SQL generated by the model against the expected SQL from `slm_exploration.xlsx`, highlighting exact matches and discrepancies.

### How it Works

The script reads questions from `slm_exploration.xlsx`, sends them one by one to the API endpoint, and meticulously records the entire event stream. The `QuerifaiTracker` class is a stateful processor that manages the complexity of tracking multiple, potentially nested, `querifai` sessions. After each question, it saves the progressively growing results to the output Excel file, ensuring no data is lost if the run is interrupted.
